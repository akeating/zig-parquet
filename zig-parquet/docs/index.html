zig-parquet API Docs

Use the Zig API when you want the native `parquet` module with comptime-driven row types and direct access to the library internals. Use the C API when you need a stable ABI for C, C++, or another FFI consumer.

Zig API

Best fit for Zig applications using `Reader`, `Writer`, `RowReader(T)`, `RowWriter(T)`, and `DynamicReader`.

  • Browse Zig API reference
  • See the basic Zig examples

C API

Best fit for embedding `zig-parquet` behind a C ABI, including typed read and write helpers plus nested schema builders.

  • Browse C API reference
  • Inspect the installed public header
