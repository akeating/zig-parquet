# Parquet Test Files

Canonical Parquet test files generated by PyArrow (which wraps the reference C++ implementation). These files are used to validate Parquet reader implementations.

## Quick Start

```bash
# Install dependencies (uses uv)
uv sync

# Regenerate all files
uv run python generate.py

# Regenerate only P0 (basic) files
uv run python generate.py --p0
```

## File Structure

```
test-files-arrow/
├── basic/                    # P0: Core functionality
├── encodings/                # Encoding variations (dictionary, delta, byte_stream_split)
├── compression/              # Compression codecs (zstd, gzip, snappy, lz4, brotli)
├── structure/                # File structure (multiple row groups)
├── edge_cases/               # Edge cases (empty, single value, large strings)
├── logical_types/            # Logical type annotations
├── nested/                   # Nested types (list, map, struct)
├── multipage/                # Multi-page column chunks
├── interop/                  # Interop: files written by zig-parquet, validated by PyArrow
├── manifest.json             # Expected metadata for validation
├── generate.py               # Regeneration script
└── test_interop.py           # PyArrow validation of interop/ files
```

Most directories contain **generated** test files (PyArrow writes, Zig reads) that are
gitignored and recreated by `generate.py`. The `interop/` directory is the opposite:
files **written by zig-parquet** during `zig build test` and committed to git, then
validated by `test_interop.py` to confirm cross-implementation compatibility.

## Priority Levels

### P0: MVP (start here)

| File | Tests |
|------|-------|
| `basic/basic_types_plain_uncompressed.parquet` | All primitive types (bool, int32, int64, float, double, string, binary), PLAIN encoding, no compression |
| `basic/boundary_values.parquet` | Integer min/max values for all integer types (INT8 through UINT64) |
| `basic/nulls_and_empties.parquet` | Null values, empty strings, mixed null patterns |

### P1: Essential Features

| File | Tests |
|------|-------|
| `encodings/dictionary_low_cardinality.parquet` | DICTIONARY encoding with few unique values |
| `encodings/dictionary_high_cardinality.parquet` | DICTIONARY encoding with many unique values |
| `compression/compression_zstd.parquet` | zstd decompression |
| `compression/compression_gzip.parquet` | gzip decompression |
| `compression/compression_snappy.parquet` | Snappy decompression |
| `structure/multiple_row_groups.parquet` | Row group iteration (10 row groups) |

### Edge Cases

| File | Tests |
|------|-------|
| `edge_cases/single_value.parquet` | Minimal valid file (1 row, 1 column) |
| `edge_cases/empty_table.parquet` | Valid file with schema but no rows |
| `edge_cases/large_strings.parquet` | Large string values (100KB each) |

### Logical Types (Core)

| File | Tests |
|------|-------|
| `logical_types/string.parquet` | STRING logical type (UTF8 annotation on BYTE_ARRAY) |
| `logical_types/date.parquet` | DATE logical type (days since epoch as INT32) |
| `logical_types/timestamp.parquet` | TIMESTAMP logical type (millis/micros/nanos, with/without UTC) |
| `logical_types/time.parquet` | TIME logical type (millis/micros/nanos) |

### Logical Types (Extended)

| File | Tests |
|------|-------|
| `logical_types/decimal.parquet` | DECIMAL with precision 9/18/38 (INT32, INT64, FIXED_LEN_BYTE_ARRAY) |
| `logical_types/uuid.parquet` | UUID (FIXED_LEN_BYTE_ARRAY(16)) |
| `logical_types/int_types.parquet` | INT8/16/32/64 and UINT8/16/32/64 |
| `logical_types/float16.parquet` | FLOAT16 (IEEE 754 half-precision) |
| `logical_types/enum.parquet` | ENUM (dictionary-encoded strings) |
| `logical_types/json.parquet` | JSON (large string) |

## Manifest

`manifest.json` contains expected metadata for each file:

```json
{
  "files": {
    "basic/basic_types_plain_uncompressed.parquet": {
      "sha256": "...",
      "num_rows": 5,
      "num_columns": 7,
      "num_row_groups": 1,
      "schema": "...",
      "columns": [...]
    }
  }
}
```

Use this for validation:
- Verify row/column counts match
- Verify SHA256 hash (for byte-exact comparison)
- Check schema matches expected types

## Validation Example

```python
import pyarrow.parquet as pq

# Read and verify a file
meta = pq.read_metadata("basic/basic_types_plain_uncompressed.parquet")
assert meta.num_rows == 5
assert meta.num_columns == 7

# Read actual data
table = pq.read_table("basic/basic_types_plain_uncompressed.parquet")
print(table.to_pandas())
```

## Generation Notes

- All files use `data_page_version="1.0"` for simplicity
- PLAIN encoding files use `use_dictionary=False`
- Compression files use the same source data for comparison
- Generated with PyArrow (wraps parquet-cpp reference implementation)
