# AGENTS.md

## Project Overview

zig-parquet is a native Parquet library written in Zig 0.15.2. It provides read/write support for Parquet files, a C ABI, and a CLI tool (`pqi`). Read `README.md` for the full feature list and API examples.

## Project Structure

- `zig-parquet/` - Core Parquet library
- `cli/` - `pqi` CLI tool
- `test-files-arrow/` - Test files generated by Python (uses uv)
- `test-files-wild/` - Real-world Parquet files for compatibility testing

### Key Files

- `zig-parquet/src/lib.zig` - Library exports (package root)
- `zig-parquet/src/core/dynamic_reader.zig` - DynamicReader (schema-agnostic row reading)
- `zig-parquet/src/core/column_decoder.zig` - Column value decoding (encodings, levels, types)
- `zig-parquet/src/core/writer.zig` - High-level typed Parquet writer
- `zig-parquet/src/core/dynamic_writer.zig` - DynamicWriter (runtime-schema row writing)
- `zig-parquet/src/core/format.zig` - Parquet format types (re-exports from format/)
- `zig-parquet/src/core/safe.zig` - Safe casting/slicing helpers for untrusted data
- `zig-parquet/src/core/arrow_batch.zig` - Arrow C Data Interface bridge
- `zig-parquet/src/core/schema.zig` - Recursive schema for nested types + logical types
- `cli/src/main.zig` - pqi CLI entry point

### Documentation

- `ROADMAP.md` - Development phases and progress
- `zig-parquet/HARDENING.md` - Hardening guidelines
- `test-files-wild/README.md` - Test file validation status

## Build & Test

```bash
cd zig-parquet && zig build test                  # Run library tests (all codecs)
cd zig-parquet && zig build test -Dcodecs=none    # Run non-compression tests only
cd zig-parquet && zig build test -Dcodecs=zstd    # Run with C zstd only
cd zig-parquet && zig build test -Dcodecs=zig-only # Run with pure Zig codecs only
cd cli && zig build                               # Build pqi CLI
cd cli && ./validate-wild.sh                      # Validate against wild test files (failures only)
cd cli && ./validate-wild.sh --all                # Show all results
```

Zig 0.15 does NOT support `--test-filter`. Tests are discovered automatically via `@import`.

## Zig 0.15.2 — Key API Differences from 0.14

Always search the codebase for existing patterns before writing new code.

### ArrayList / ArrayListUnmanaged

```zig
var list: std.ArrayList(T) = .empty;     // NOT .init(allocator)
try list.append(allocator, item);         // allocator passed to ALL methods
const slice = try list.toOwnedSlice(allocator);
list.deinit(allocator);
```

### File I/O (Streaming Writers)

```zig
var buffer: [8192]u8 = undefined;
var file_writer = file.writerStreaming(&buffer);
const writer: *std.Io.Writer = &file_writer.interface;
try writer.writeAll(data);
try file_writer.interface.flush();        // always flush
```

### Memory-backed Writers

```zig
var out: std.io.Writer.Allocating = .init(allocator);
defer out.deinit();
try out.writer.writeAll(data);
const owned = try out.toOwnedSlice();
```

Two incompatible writer interfaces exist in 0.15:
- `*std.io.Writer` (new) — required by `std.json.Stringify`, file I/O
- `std.io.GenericWriter` (old) — returned by `ArrayList.writer()`

If a stdlib function needs `*std.io.Writer`, use `std.io.Writer.Allocating`.

### stdout / stderr

```zig
var buf: [256]u8 = undefined;
var stdout_writer = std.fs.File.stdout().writer(&buf);
const stdout = &stdout_writer.interface;
try stdout.writeAll("message\n");
```

### JSON Serialization

```zig
// Serialize
var out: std.io.Writer.Allocating = .init(allocator);
errdefer out.deinit();
try std.json.Stringify.value(data, .{}, &out.writer);
return out.toOwnedSlice();

// Parse
var parsed = try std.json.parseFromSlice(MyData, allocator, json, .{});
defer parsed.deinit();
```

### ArenaAllocator

```zig
_ = arena.reset(.free_all);           // NOT arena.reset()
_ = arena.reset(.retain_capacity);
```

### Build Options (Conditional Compilation)

Compression codecs are controlled via `-Dcodecs=` (default: all). Values: `all`, `none`, `zig-only`, or comma-separated list of: `zstd,zig-zstd,snappy,gzip,lz4,brotli`.

```zig
// Stable codecs at top level, experimental behind namespace
pub const zstd = if (build_options.enable_zstd) @import("zstd.zig") else {};
pub const snappy = if (build_options.enable_snappy) @import("snappy.zig") else {};

pub const experimental = struct {
    pub const zig_zstd = if (build_options.enable_zig_zstd) @import("zig_zstd.zig") else {};
};
```

Build option flags:
- Implementation-specific: `enable_zstd` (C), `enable_zig_zstd` (Zig), `enable_snappy`, `enable_gzip`, `enable_lz4`, `enable_brotli`
- Category: `supports_zstd` (true if either C or Zig zstd is enabled)

In tests, guard codec-specific tests with `supports_*` flags:

```zig
test "my zstd test" {
    if (!build_options.supports_zstd) return;
    // ...
}
```

### WASM Targets

- `export fn` does NOT use `callconv(.C)` on WASM — it won't compile.
- `wasm32-freestanding` won't work with `std.posix`, `std.Thread`, or `std.debug.print`. Use `wasm32-wasi`.

### Before Writing Code

1. Search the codebase for similar patterns first.
2. Compile incrementally — test after every 20-30 lines, not 200.
3. Read compiler errors carefully — Zig's errors are precise.
4. Never call deprecated methods — check for newer alternatives.

## Hardening Requirements

Both reader and writer handle **untrusted data**. Code must return errors instead of panicking on malformed input. See `zig-parquet/HARDENING.md` for full details.

### Never Use `@intCast` on External Data

Use `safe.zig` for all external data casting:

```zig
const safe = @import("safe.zig");

// BAD — panics if value doesn't fit
const size: usize = @intCast(header.page_size);

// GOOD — returns error if value is negative or out of range
const size = try safe.cast(header.page_size);
const count = try safe.castTo(u32, header.num_values);
```

### `catch unreachable` Requirements

When a value is mathematically guaranteed to fit, use `catch unreachable` but MUST include an inline comment explaining why:

```zig
const bit_idx: u3 = safe.castTo(u3, i % 8) catch unreachable; // i % 8 is strictly 0-7

if (value.len > std.math.maxInt(i32)) return error.ValueTooLarge;
const len = safe.castTo(i32, value.len) catch unreachable; // Checked against maxInt(i32) above
```

### Always Bounds-Check Before Slicing or Indexing

```zig
// BAD — panics if out of bounds
const value = data[offset..][0..length];

// GOOD
const value = try safe.slice(data, offset, length);
const item = try safe.get(T, data, index);
```

### Use Checked Arithmetic for External Values

```zig
// BAD — can underflow silently
const values_size = total_size - header_size;

// GOOD
if (total_size < header_size) return error.InvalidSize;
const values_size = total_size - header_size;

// GOOD
const total = std.math.add(usize, a, b) catch return error.Overflow;
```

### Return Context-Aware Errors

Use specific errors (`error.InvalidBitWidth`, `error.EndOfData`, `error.InvalidPageSize`) instead of generic `error.InvalidData`.

### Validate at API Boundaries

```zig
pub fn writeColumn(values: []const T) !void {
    if (values.len > std.math.maxInt(i32)) return error.TooManyValues;
    // ... proceed with encoding
}
```

### Common Safe Casting Helpers

- `safePageSize(i32) -> error{InvalidPageSize}!usize`
- `safeRowCount(i64) -> error{InvalidRowCount}!usize`
- `extractBitWidth([]const u8, usize) -> error{InvalidBitWidth, EndOfData}!u5`
- `safeTypeLength(?i32) -> error{InvalidTypeLength}!usize`
