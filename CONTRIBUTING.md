# Contributing

Contributions are welcome! This document covers how to get started.

## Development Setup

**Requirements:**
- Zig 0.15.2
- C compiler (for compression libraries)
- C++ compiler (for Snappy)

```bash
# Build and test the library
cd zig-parquet
zig build test

# Build the CLI
cd cli
zig build
```

## Test Files

Test files in `test-files-arrow/` are generated by Python scripts. After cloning, generate them with:

```bash
cd test-files-arrow
uv run generate.py
```

Real-world test files live in `test-files-wild/`. Run validation against them:

```bash
cd cli
./validate-wild.sh        # show failures only
./validate-wild.sh --all  # show all results
```

## Code Style

- Follow existing patterns in the codebase
- Use `safe.zig` for all external data casting — never use `@intCast` on untrusted data
- Bounds-check before slicing or indexing with `safe.slice` and `safe.get`
- Use checked arithmetic (`std.math.add`, explicit underflow guards) for external values
- Return specific error names (`error.InvalidBitWidth`, not `error.InvalidData`)
- Validate user input at API boundaries

See `zig-parquet/HARDENING.md` for the full hardening guidelines.

## Submitting Changes

1. Fork the repository
2. Create a branch for your change
3. Make sure `zig build test` passes in `zig-parquet/`
4. Submit a pull request with a clear description of what and why
